git 使用

从github 复制的

实际使用的情况

github 新建仓库

本地仓库初始化(init)

git init

添加所有文件到暂存区

git add *

将暂存区的文件提交到仓库

git commit -m “第一次提交”

将主分支重命名为 ：main

git branch -M main

添加远程仓库，将本地仓库与远程仓库建立联系

git remote add origin git@github.com:99fukong/2021-11-28-wangye.git

将分支main 推送远程仓库

git push -u origin main

查看提交状态

git log （提交状态）

查看git 仓库状态(工作区，暂存区)

git status

修改文件后，使用git status，显示的是工作区状态

红色表示，改文件处于工作区

使用 git diff 查看工作区所改动的内容

git diff

把工作区的改动文件（shouye.html）添加到暂存区

第一种：

  • git add shouye.html

第二种:

  • git add -u
  • -u : 表示所有已提交且改动了的文件

添加暂存区后，使用git status 查看仓库状态

git status

颜色由红色变为绿色，表示该文件已经添加到暂存区

将暂存区的文件提交到仓库

git commit -m “第二次提交”

查看提交日志（log）

git log

推送到远程仓库

git push origin main

撤销工作区改动的方法

  • 第一种：
    • 使用vscode界面
  • 第二种：
    • 使用 git 命令：git checkout xxxxx

撤销已经提交到暂存区的改动

HEAD ：最近一次提交的状态

HEAD^ ：上一次提交的状态

HEAD^^ ：上上一次提交的状态

git reset –soft HEAD : 回到HEAD 版本，将撤销的内容放入到暂存区

git reset –mixed HEAD：回到HEAD 版本，将撤销的内容放入工作区

git reset –hard HEAD: 回到HEAD 版本，将三个区内容全部丢弃

git reset –soft HEAD^ : 回到HEAD^ 版本，将撤销的内容放入到暂存区

git reset –mixed HEAD^：回到HEAD^版本，将撤销的内容放入工作区

git reset –hard HEAD^: 回到HEAD^版本，将三个区内容全部丢弃

把远程分支合并到本地

git pull

  • 如果有冲突还需要解决冲突（高级，不管）
